# i18n Guidelines

## Explicit IDs (mandatory)

Every translation must use a manually assigned, semantically rich `id`. Never rely on Lingui's auto-generated hashes.

IDs use **dot-namespaced** format: `{page}.{section}.{element}`

```
home.hero.title
home.hero.subtitle
dashboard.settings.notificationsLabel
pricing.pro.cta
common.nav.signIn
```

## Comments (mandatory)

Every translation must include a `comment` that gives translators enough context to produce an accurate translation. Describe where the string appears, what it refers to, or any constraints (e.g. character limits).

## Syntax

### JSX — `<Trans>`

```tsx
<Trans id="home.hero.title" comment="Main heading on the landing page">
  Find your next opportunity
</Trans>
```

### JavaScript — `t()`

```ts
t({
  id: "home.hero.title",
  comment: "Main heading on the landing page",
  message: "Find your next opportunity",
})
```

### Plurals — `plural()`

```ts
plural(count, {
  id: "dashboard.feed.jobCount",
  comment: "Number of matching jobs shown above the feed",
  one: "# job found",
  other: "# jobs found",
})
```

## Naming conventions

| Segment   | Rule                                  | Examples                          |
| --------- | ------------------------------------- | --------------------------------- |
| page      | Route or feature name                 | `home`, `dashboard`, `pricing`    |
| section   | Logical area within the page          | `hero`, `features`, `settings`    |
| element   | Specific piece of text                | `title`, `cta`, `description`     |
| `common.` | Prefix for strings reused across pages | `common.nav.signIn`, `common.footer.copyright` |

Keep IDs **lowercase**, use **camelCase** only for the element segment when needed (`notificationsLabel`).

## What not to do

```tsx
// Missing id — will get an auto-generated hash
<Trans>Find your next opportunity</Trans>

// Missing comment — translators lack context
<Trans id="home.hero.title">Find your next opportunity</Trans>

// Vague id — not semantically useful
<Trans id="text1" comment="...">Find your next opportunity</Trans>
```
